Algoritmo


Es una lista finita y ordenada de pasos necesarios para llegar a la solución de un problema específico

PARTES DEL ALGORITMO

Entradas

Son los valores obtenidos del problema que representan sl mundo real. Pueden ser cero a más entradas

Proceso

Descripción de las operaciones que se llevarán a cabo con las entradas anteriores

Salidas

La salida son los valores resultantes a transformar los valores de la entrada. Pueden ser una o más salidas

¿Cómo realizar un Algoritmo?


IDENTIFICAR EL PROBLEMA

Es el enunciado del problema, se pretende que deba ser claro y completo.

ANALIZAR EL PROBLEMA

Leer el enunciado cuantas veces sea necesario con el objetivo de saber que se requiere como solución.

HACER EL ALGORITMO QUE DA SOLUCIÓN AL PROBLEMA

Sin tener dudas de lo que se requiere como solución al problema, procedemos a escribir el algoritmo.

HACER EL ANÁLISIS DE LOS DATOS

Hacer las partes que conforman el Algoritmo: - Entrada de datos - Proceso de los datos - Salida de datos

Variable

Es una palabra o identificador creado por el programador para asignarle valores.

¿Cómo crear el nombre de una variable

El identificador o nombre de variable, debe constar solo de letras y números, comenzando siempre por una letra.

Criterio importante para el nombre de una variable

Elige el nombre de tus variables de tal forma que se sobreentienda el valor que va a representar de acuerdo al problema que se está resolviendo.

DIAGRAMAS DE FLUJO

El Diagrama de Flujo es la representación graficosimbólica del algoritmo, nos indica al flujo lógico de cada paso definido.

SIMBOLOGÍA BÁSICA DE DIAGRAMAS DE FLUJO

El óvalo indica el inicio y el final del diagrama de flujo. Indicar su uso con la palabra a su interior.

El trapecio indica recibir en una variable un dato de entrada usando el teclado. Poner la variable a su interior.

El cono horizontal indica enviar el contenido de una variable a salida, representa el monitor o pantalla. Podemos indicar que mande de salida un mensaje de texto poniéndole comilla sencilla en los extremos.

El rectángulo partido indica enviar el contenido de una variable salida, representa la impresora o papel. Podemos indicar que mande de salida un mensaje de texto poniéndole comillas sencillas en los extremos.

El romboide inidica recibir en una variable un dato de entrada usando el teclado o enviar un dato contenido en una variable en la salida, comúnmente el monitor o pantalla. Al usarlo debemos indicar a su interior la palabra LEER O ESCRIBIR según su uso. Podemos indicar que mande salida un mensaje de texto.

El rectángulo indica proceso de datos. Hacer un cálculo de operaciones aritméticas. Para asignar el cálculo aritmético a una variable se usa = ó <-

El pentágono para conectar segmentos de diagrama de una página a otra, usar número o letra secuencial a su interior.

El círculo para conectar segmentos de diagrama en la misma página, usar número o letra secuencial a su interior.

El mini círculo sirve para unir intersecciones de líneas de flujo.

Flecha vertical indica el flujo hacia debajo de los pasos del algoritmo.

Las flechas horizontales indican el flujo hacia la zquierda o hacia la derecha.


PROPIEDADES DE DIAGRAMA DE FLUJO

1.- Un diagrama de flujo siempre se realiza de arriba hacia abajo y de izquierda a derecha.

2.- Las flechas de flujo siempre tocan el símbolo superior y al símolo inferior.

3.- Las flechas de flujo nunca se cruzan, usar mini círculo para unir un flujo.

PSEUDOCÓDIGO

Es la representación narrativa de cada paso del algoritmo; esta narrativa obedece a la lengua nativa del programador.

El pseudocódigo utiliza palabras que indican el proceso a realizar:

- INICIO

- FINAL

Respectivamente indican el inicio del pseudocódigo y final del pseudocódigo

Palabras a utilizar:

Leer variable

Palabras a utilizar:

Escribir "mensaje"

Escribir "mensaje", variable

Palabras a utilizar:

Para los cálculos aritméticos no existe una palabra específica

LENGUAJE DE PROGRAMACIÓN

Es un conjunto de palabras con cierta base sintáctica y semántica, para construir instrucciones que forman un programa de computadora

Comprende un conjunto de palabras reservadas y propias. Las cuales tienen su base sintáctica de uso y su base semántica de forma.

ALGORITMOS CONDICIONALES

Hemos llegado a un punto de la resolución de un problema, donde existan diferentescaminos con accioneso pasos que puedan llegar a ejecutarse.

PERO SOLO UN CAMINO SE PUEDE EJECUTAR EN ALGÚN MOMENTO

Esquema:

Debe existir: una pregunta

Que termine: una condición para algún dato

Y evaluarla con: un valor de verdad

La condición para algún dato se expresa como una proposición u oración que nos lleve a valores como: sí ó no, v óf, verdadero ó falso, true o false, on u off, 1 ó 0.

El rombo sirve para evaluar su condición, al evaluarla se tiene un valor de verdad de dos valores posibles:

El valor de la condición puede ser verdadero

El valor de la condición puede ser falso

La ejecución del flujo sigue en el lado que distinga el verdadero si la condición es verdadera.

La ejecución del flujo sigue en el lado que distinga el falso si la condición es falso.

Si la condición SI se cumple

Si la condición NO se cumple

La ejecución del flujo sigue en el lado que distinga el SI si la condición si se cumple.

La ejecución del flujo sigue en el lado que distinga el NO si la condición no se cumple. Además de poder intercambiar de lugar:

El V ó F

Ó el SI ó NO

También podemos cambiar de lugar las líneas de flujo

OPERADORES CONDICIONALES O COMPARATIVOS

<= Menor o igual que

< Mayor que

>= Mayor o igual que

!= Diferente a

PSEUDOCÓDIGO CONDICIONAL VS DIAGRAMA DE FLUJO

Si condición Entonces

paso x

paso x+1

Sino

paso n

paso n+1

FinSi


La palabra Si-Entonces es el flujo verdadero "SI" del rombo, la palabra Sino es el flujo falso "NO" del rombo y se debe terminar con FinSi.

ACLARACIÓN IMPORTANTE

Un rombo puede contener en sus flujos SI y NO, más rombos y estos a su vez también. Por lo mismo un Si-Entonces solo o con el Sino pueden contener más Si-Entonces con su Sino o solo el Si-Entonces y estos a su vez también.

POR SU CONTENIDO

Variables Numéricas

Son aquellas en las cuales se almacenan valores numéricos, positivos o negativos, enteros o flotantes (números con punto decimal).

Variables Lógicas

Son aquellas que solo pueden tener dos valores (verdadero ó falso, 0 ó 1, si ó no, cierto ó falso, TRUE O FALSE, ON o OFF) estos valores representan el resultado de una comparación entre un dato y otro dato.

Variables Alfanuméricas

Son aquellas en las cuales se almacenan valores o información compuesta por letras y/ó números y/ó carácteres especiales(carácteres alfanuméricos).

NOTA

Se le llaman carácteres alfanuméricos a las letras, números y carácteres especiales que observas en tu teclado y son imprimibles.
